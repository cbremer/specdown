# Specdown iOS

Swift + WKWebView app wrapping the shared `markdown-viewer/` web codebase.

## Prerequisites

- Xcode 16+ (iOS 16 SDK)
- [XcodeGen](https://github.com/yonaskolb/XcodeGen): `brew install xcodegen`

## Getting Started

```bash
# From the ios/ directory:
xcodegen generate        # Creates SpecDown.xcodeproj from project.yml
open SpecDown.xcodeproj  # Opens in Xcode
```

Then select an iPhone or iPad simulator and press Run.

## Structure

```
ios/
├── project.yml              # XcodeGen spec (source of truth for project config)
├── SpecDown/
│   ├── App.swift            # @main entry point
│   ├── ContentView.swift    # Root SwiftUI view
│   ├── WebView.swift        # UIViewRepresentable wrapping WKWebView
│   ├── WebBridge.swift      # JS ↔ Swift IPC (WKScriptMessageHandler)
│   └── Assets.xcassets/     # App icon, accent color
└── SpecDown.xcodeproj/      # Generated by XcodeGen — do not edit manually
```

## Web Assets

The `markdown-viewer/` directory at the repo root is bundled into the app as a folder reference. Any changes to the web viewer are automatically picked up on the next build — no separate copy step needed.

## Why XcodeGen?

The `.xcodeproj` is generated from `project.yml` and is **not committed to git**. This avoids notoriously messy `.pbxproj` merge conflicts. Always run `xcodegen generate` after pulling changes to `project.yml`.
